# MarketingLead Enumerations - AI/Agent Context

## Purpose

This folder contains enumeration types for marketing lead categorization and classification. These enums map to Lookup table values in the database and provide type-safe references throughout the codebase.

**Pattern**: Integer-backed enums with explicit database ID values (not zero-indexed).

---

## Enum Inventory

### 1. **CallType** (Call direction/origin)
**Purpose**: Classify the direction and type of phone call.

```csharp
public enum CallType
{
    Inbound = 140,
    Outbound = 141,
    Other = 142,
    ClickTo = 143,
    AgentOutbound = 144
}
```

**Values**:
- **Inbound (140)**: Customer-initiated inbound call to tracking number
  - **Usage**: Primary lead type - customer calls business
  - **Source**: Web clicks, phone listings, advertisements
  - **Billing**: Typically charged to franchisee as lead

- **Outbound (141)**: Business-initiated outbound call to customer
  - **Usage**: Follow-up calls, callbacks, proactive outreach
  - **Source**: Internal CRM, follow-up queues
  - **Billing**: Not typically charged as lead

- **Other (142)**: Uncategorized or system calls
  - **Usage**: Test calls, system checks, unclassified
  - **Source**: Various

- **ClickTo (143)**: Click-to-call from website
  - **Usage**: Customer clicks button to initiate call
  - **Source**: Website click-to-call widgets
  - **Billing**: High-value lead (engaged customer)

- **AgentOutbound (144)**: Agent-initiated outbound call
  - **Usage**: Sales agent or CSR making outbound call
  - **Source**: Call center, sales team
  - **Billing**: Not a lead (business-initiated)

**Database Mapping**: Foreign key to `Lookup` table (CallTypeId = 140-144)

**Business Logic**:
- Inbound and ClickTo calls count as qualified leads
- Outbound and AgentOutbound do not count as new leads
- Used in reporting to filter lead sources vs follow-ups

### 2. **RoutingNumberCategory** (Marketing channel classification)
**Purpose**: Categorize phone tracking numbers by marketing channel/source.

```csharp
public enum RoutingNumberCategory
{
    PrintMedia = 191,
    PhoneWebLocal = 192,
    PhoneWebNational = 193,
    BusinessDirectories = 194,
    WRAPVan = 195
}
```

**Values**:
- **PrintMedia (191)**: Print advertisements
  - **Examples**: Newspaper ads, magazine ads, direct mail
  - **Tracking**: Each print campaign gets unique number
  - **ROI Analysis**: Measures print ad effectiveness

- **PhoneWebLocal (192)**: Local franchisee website traffic
  - **Examples**: Franchisee-specific landing pages, local SEO
  - **Tracking**: Tracks calls from franchisee's local site
  - **Attribution**: Credits franchisee's local marketing efforts

- **PhoneWebNational (193)**: National corporate website traffic
  - **Examples**: marblelife.com main site, national PPC campaigns
  - **Tracking**: Tracks calls from corporate site
  - **Attribution**: Credits corporate marketing spend

- **BusinessDirectories (194)**: Online business directories
  - **Examples**: Google My Business, Yelp, Yellow Pages
  - **Tracking**: Each directory listing gets unique number
  - **ROI Analysis**: Measures directory listing effectiveness

- **WRAPVan (195)**: Vehicle wrap advertising
  - **Examples**: Phone numbers on service vans, branded vehicles
  - **Tracking**: "Call the number on our van" campaigns
  - **ROI Analysis**: Measures mobile billboard effectiveness

**Database Mapping**: Foreign key to `Lookup` table (CategoryId = 191-195)

**Usage in RoutingNumber Entity**:
```csharp
public class RoutingNumber
{
    public long? CategoryId { get; set; }  // Maps to RoutingNumberCategory
    [ForeignKey("CategoryId")]
    public virtual Lookup Category { get; set; }
}
```

**Business Logic**:
- Used in performance reports to compare channel effectiveness
- PhoneWebLocal vs PhoneWebNational split attributes leads correctly
- Determines which budget (franchisee vs corporate) pays for lead

### 3. **TagType** (Lead attribution)
**Purpose**: High-level categorization of lead origin (franchisee vs corporate).

```csharp
public enum TagType
{
    FranchiseDirect = 1,
    National = 2
}
```

**Values**:
- **FranchiseDirect (1)**: Lead generated by franchisee's direct marketing
  - **Sources**: Local SEO, franchisee PPC, local ads, print media
  - **Billing**: Franchisee pays lead cost
  - **Ownership**: Franchisee owns the lead
  - **Reporting**: Shows in franchisee's marketing ROI

- **National (2)**: Lead generated by corporate national marketing
  - **Sources**: National PPC, TV ads, national SEO (marblelife.com)
  - **Billing**: Corporate pays lead cost
  - **Ownership**: Routed to franchisee based on geography
  - **Reporting**: Shows in corporate marketing ROI

**Database Mapping**: Foreign key to `Tag` table (TagId = 1-2)

**Usage in Entities**:
```csharp
// MarketingLeadCallDetail
public long TagId { get; set; }
[ForeignKey("TagId")]
public virtual Tag Tag { get; set; }

// RoutingNumber
public long? TagId { get; set; }
[ForeignKey("TagId")]
public virtual Tag Tag { get; set; }
```

**Business Logic**:
- Determines who pays for the lead (franchisee vs corporate)
- Critical for P&L allocation in franchise model
- Used in performance reports to show marketing source effectiveness
- Routing numbers inherit tag from their configuration

**Example Scenario**:
```
RoutingNumber: 555-1234
  PhoneLabel: "Local SEO"
  TagId: FranchiseDirect (1)
  FranchiseeId: 123

When call comes to 555-1234:
  → MarketingLeadCallDetail.TagId = 1 (inherited from RoutingNumber)
  → Franchisee 123 pays for lead
  → Appears in franchisee's "Direct Marketing" report
```

### 4. **PerformanceParameter** (Reporting cost categories)
**Purpose**: Categorize marketing spend for performance analysis.

```csharp
public enum PerformanceParameter
{
    SEOCOST = 219,
    PPCSPEND = 220
}
```

**Values**:
- **SEOCOST (219)**: Search Engine Optimization expenses
  - **Examples**: SEO agency fees, content creation, link building
  - **Tracking**: Monthly or annual SEO budget
  - **ROI Metric**: Organic leads generated / SEO cost

- **PPCSPEND (220)**: Pay-Per-Click advertising spend
  - **Examples**: Google Ads, Bing Ads, social media ads
  - **Tracking**: Total ad spend for period
  - **ROI Metric**: PPC leads generated / PPC spend

**Database Mapping**: Foreign key to `Lookup` table (likely used in reporting queries)

**Usage Pattern** (inferred from naming):
```csharp
// Likely used in performance reports
var seoCost = _lookupRepository.Table
    .FirstOrDefault(x => x.Id == (long)PerformanceParameter.SEOCOST);

var ppcSpend = _lookupRepository.Table
    .FirstOrDefault(x => x.Id == (long)PerformanceParameter.PPCSPEND);

// Calculate ROI
var roi = (leadsGenerated * avgLeadValue) / ppcSpend.Value;
```

**Business Logic**:
- Used in LeadPerformance reports to show cost per lead
- Allows tracking marketing spend vs lead volume
- Enables ROI calculations (revenue from leads / marketing spend)

    call.CallTypeId == (long)CallType.ClickTo)

    // Bill corporate

## Best Practices

### 1. Always Cast to long for Database Queries
```csharp
// Correct
.Where(x => x.CallTypeId == (long)CallType.Inbound)

// Incorrect (may cause conversion errors)
.Where(x => x.CallTypeId == (int)CallType.Inbound)
```

### 2. Use Enum Values, Not Magic Numbers
```csharp
// Good - readable and type-safe
if (call.CallTypeId == (long)CallType.Inbound) { }

// Bad - magic number, hard to understand
if (call.CallTypeId == 140) { }
```

### 3. Handle Nullable Enum References
```csharp
// RoutingNumber.CategoryId is nullable
if (routingNumber.CategoryId == (long)RoutingNumberCategory.PhoneWebLocal)
{
    // Handle local web leads
}
```

### 4. Document New Enum Values
If adding new values:
1. Add to enum with next available ID in sequence
2. Add corresponding Lookup/Tag table record
3. Update factory methods to handle new value
4. Update report queries if filtering by enum

## Reporting Usage

### Lead Source Reports
```csharp
// Group by CallType
var byCallType = calls
    .GroupBy(x => x.CallTypeId)
    .Select(g => new {
        CallType = g.Key == (long)CallType.Inbound ? "Inbound" : 
                   g.Key == (long)CallType.ClickTo ? "Click-to-Call" : "Other",
        Count = g.Count()
    });

// Group by TagType
var byTag = calls
    .GroupBy(x => x.TagId)
    .Select(g => new {
        Source = g.Key == (long)TagType.FranchiseDirect ? "Franchisee Direct" : "National",
        Count = g.Count()
    });
```

### Marketing Channel Reports
```csharp
// ROI by category
var byCategory = routingNumbers
    .GroupBy(x => x.CategoryId)
    .Select(g => new {
        Category = (RoutingNumberCategory)g.Key.Value,
        Numbers = g.Count(),
        Calls = _callRepo.Table.Count(c => c.RoutingNumber.CategoryId == g.Key)
    });
```
